# Signals

Signal endpoints provide access to current and historical trading signals generated by strategies, including those that pass confidence and edge thresholds.

## Endpoints

| Method | Path | Description | Auth Required |
|--------|------|-------------|---------------|
| GET | `/api/signals` | Current trading signals generated by scanning active markets | Yes (Admin) |
| GET | `/api/signals/actionable` | Signals that pass configured edge and confidence thresholds | Yes (Admin) |
| GET | `/api/signals/history` | Historical signal database records with outcome data | Yes (Admin) |
| GET | `/api/calibration` | Signal calibration data and accuracy analysis | Yes (Admin) |

## Signal Object Structure

A signal represents a trading opportunity identified by a strategy.

Example Response (**GET `/api/signals`**):
```json
[
  {
    "market_ticker": "btc-5m-2026-04-13-10-30",
    "market_title": "BTC 5m - 2026-04-13 10:30",
    "platform": "polymarket",
    "direction": "up",
    "model_probability": 0.65,
    "market_probability": 0.58,
    "edge": 0.07,
    "confidence": 0.82,
    "suggested_size": 150.0,
    "reasoning": "RSI at 42, strong volume increase, support at 65000",
    "timestamp": "2026-04-13T10:25:00Z",
    "category": "crypto",
    "event_slug": "btc-5m-2026-04-13-10-30",
    "btc_price": 65123.45,
    "btc_change_24h": 0.024,
    "window_end": "2026-04-13T10:30:00Z",
    "actionable": true
  }
]
```

## Signal Calibration

Calibration data compares predicted model probabilities against actual outcomes to track accuracy over time.

**GET `/api/calibration`**

Example Response:
```json
{
  "buckets": [
    {
      "bucket": "60-65%",
      "predicted_avg": 0.625,
      "actual_rate": 0.61,
      "count": 120
    }
  ],
  "summary": {
    "total_signals": 1500,
    "total_with_outcome": 1400,
    "accuracy": 0.63,
    "avg_predicted_edge": 0.045,
    "avg_actual_edge": 0.041,
    "brier_score": 0.22
  }
}
```

:::info
The **Brier Score** measures the accuracy of probability forecasts. A lower score indicates better calibration, with 0.0 being a perfect prediction and 0.25 being as accurate as random guessing.
:::

## Signal History

The signal history endpoint provides outcome data for past predictions, including settlement values and correctness.

**GET `/api/signals/history`**

Example Response:
```json
{
  "items": [
    {
      "id": 150,
      "market_ticker": "btc-5m-2026-04-13-10-00",
      "platform": "polymarket",
      "market_type": "btc",
      "timestamp": "2026-04-13T09:55:00Z",
      "direction": "up",
      "model_probability": 0.72,
      "market_probability": 0.65,
      "edge": 0.07,
      "confidence": 0.9,
      "actual_outcome": "up",
      "outcome_correct": true,
      "settlement_value": 1.0,
      "settled_at": "2026-04-13T10:01:00Z"
    }
  ],
  "total": 1500
}
```
