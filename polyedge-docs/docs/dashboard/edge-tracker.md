# Edge Tracker

The Edge Tracker provides an interface for monitoring the discovery and performance of the bot's alpha discovery engines. It gives users a detailed view of how each parallel discovery track is performing across multiple timeframes.

**Route Path:** `/edge-tracker`

## Layout

The page is organized into several key areas:

- **Header**: Contains controls for selecting the tracking timeframe (3, 7, 14, or 30 days) and a manual refresh button.
- **Tracks Grid**: A responsive grid of cards, each representing a single edge discovery engine (e.g., Real-time Scanner, Whale PNL Tracker, Commodity MR).
- **Legend and Info Panel**: A bottom-mounted footer area explaining the different edge tracks and the promotion rules for live trading.

## Features

- **Parallel Discovery**: Each track (Legacy, Real-time, Whale, Commodity) runs in isolation, providing independent alpha signals.
- **Performance Metrics**: Monitor key statistics for each track, including win rate, total PNL, and the number of signals executed versus total signals generated.
- **Execution Rate**: A visual indicator of how effectively the bot is acting on the signals generated by each discovery engine.
- **Status Tracking**: Each track is labeled with its current status (Live, Paper, or Offline).
- **Promotion Logic**: Tracks are promoted from paper to live trading once they achieve a win rate exceeding 55% with a 95% confidence interval over a 14-day period.

## Data Sources

The Edge Tracker is powered by the following API:

- **Edge Performance API**: `fetchEdgePerformance` (on demand and on timeframe change) providing metrics for all configured discovery tracks.
- **Backtest Engine**: Performance data for paper tracks is derived from the bot's internal backtest and simulation engine.
- **Live Trade API**: For live tracks, performance data is pulled from actual trade settlement records.

:::tip
A track with a high "Signal Execution" rate (>50%) and a stable win rate (>55%) is a strong candidate for promotion to live trading if it's currently in paper mode.
:::
